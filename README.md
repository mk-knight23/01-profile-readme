## Additional Features (vNext)

Six focused improvements aligned with DevTools engineering patterns:

### Keyboard Navigation
**Why added:** DevTools engineers live in keyboards. Navigation should respect that.

**What changed:** Arrow keys (‚Üë‚Üì‚Üê‚Üí) smoothly scroll between sections. Works intuitively‚Äîdown goes to next section, up goes to previous. Ignores inputs when focused on form elements.

### Project Tag Filtering
**Why added:** Filtering is a core pattern in developer tools‚Äîfrom file trees to log viewers. It lets recruiters quickly find relevant projects.

**What changed:** Added filter buttons (All, Tools, React, Node.js, TypeScript) above the project grid. Filtering is instant with smooth show/hide transitions. Uses data attributes for clean implementation.

### Keyboard Shortcuts Modal
**Why added:** Every good CLI or IDE tool has discoverable keyboard shortcuts. This is a DevTools portfolio‚Äîit should demonstrate that philosophy.

**What changed:** Press `?` anywhere to open a keyboard shortcuts modal. Shows all available navigation commands. Closes with `Esc` or clicking outside. Small touch that signals keyboard-first thinking.

### Design Decisions Section
**Why added:** Senior engineers document tradeoffs. "Why" matters more than "what."

**What changed:** Added a new section documenting three key technical decisions: vanilla over frameworks, no dark mode choice, and monorepo strategy for VIBE. Each decision includes the reasoning and explicit tradeoff.

### Scroll Progress Indicator
**Why added:** Long portfolios can feel endless. A subtle progress bar gives you a sense of position.

**What changed:** A thin 2px line at the top fills as you scroll. Minimal‚Äîdoesn't distract, just informs.

### Print-Optimized Styles
**Why added:** Recruiters often print portfolios or save as PDF for review.

**What changed:** `@media print` CSS hides navigation/footer, shows URLs next to links, ensures clean page breaks. Cmd+P produces a clean resume-style output.

### Intentionally Rejected: Search Bar
I considered adding a global search for projects. Rejected because with only 4 featured projects, filtering provides enough navigation without the complexity of a search implementation. The portfolio should remain simple and fast‚Äîprinciples I'd bring to any DevTools role.
